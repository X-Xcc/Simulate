## 1. 为什么需要微调（1min）

大家好，今天我讲最后一部分——**为什么需要微调模型，以及我们怎么做的**。

YOLOv8n-pose 已经在 COCO 数据集上训练过了，COCO 是一个大型通用数据集，里面有各种场景：街道、室内、运动场……它已经学会了"人体姿态"的基本概念。

**但监狱场景很特殊：**

- **穿统一服装**——大家都穿囚服，颜色差不多，模型很难区分谁是谁
- **光线差**——走廊灯光昏暗，摄像头角度固定，和 COCO 数据集里的场景差很远
- **行为特殊**——打架、跌倒这些行为，COCO 里标注得不够细

打个比方：YOLOv8 就像一个刚从警校毕业的新兵，基础功扎实，但没在监狱实习过。微调就是**让这个新兵在监狱里再培训一轮**，让它更适应这个特殊场景。

> 微调 = 在预训练基础上，用监狱场景的数据再训练，教模型识别特殊行为。

## 2. 微调流程 — train_pose.py（2min）

### 第一步：数据标注

用标注工具（比如 LabelMe、CVAT）在图片上标出每个人的 17 个关键点。一张图里可能有好几个人，每个人都要标。

标注完的数据按 YOLO 格式组织：

```
pose_data/
├── images/
│   ├── train/    # 训练集（80%）
│   └── val/      # 验证集（20%）
├── labels/
│   ├── train/
│   └── val/
└── pose_data.yaml   # 配置文件
```

`pose_data.yaml` 告诉 YOLO 数据在哪、多少个关键点、哪些点是可见的：

```yaml
# pose_data.yaml
path: ./pose_data
train: images/train
val: images/val

kpt_shape: [17, 3]   # 17个关键点，每个3个值(x, y, visible)

names:
  0: person
```

### 第二步：启动训练

```python
from ultralytics import YOLO

model = YOLO("yolov8n-pose.pt")  # 从预训练模型开始
results = model.train(
    data="pose_data.yaml",
    epochs=100,       # 训练100轮
    batch=8,          # 每批8张图（小显存也能跑）
    lr0=0.01,         # 初始学习率
    imgsz=640,        # 输入尺寸
    device=0          # GPU 0
)
```

### 第三步：数据增强（自动）

YOLO 训练时自动做数据增强，不需要手动处理：

| 增强方式 | 效果 |
|---------|------|
| 水平翻转 | 左右手互换，数据量翻倍 |
| 随机旋转 | 模拟摄像头不同角度 |
| 随机裁剪 | 模拟人只出现在画面局部 |
| 色彩抖动 | 模拟不同光线条件 |
| Mosaic | 4 张图拼成 1 张，增加场景复杂度 |

> 打个比方：一张照片翻转、旋转、裁剪之后，相当于变成了好几张"新照片"，让模型见过更多变体，学得更扎实。

### 第四步：输出 best.pt

训练完成后，会输出两个关键文件：
- `best.pt`——mAP 最高的权重，**用这个**
- `last.pt`——最后一轮的权重

把 `best.pt` 替换到 `models/` 目录下，重启检测程序就生效了。

**代码位置：**
- 训练脚本：`detection/train_pose.py:22`
- 数据配置：`detection/pose_data.yaml`
- 模型目录：`models/yolov8n-pose.pt`

## 3. 模型配置与加速（2min）

### Config 类 — 所有参数集中管理

我们在 `detection/yolov8_security.py:85` 里有一个 `Config` 类，所有推理参数集中管理：

```python
class Config:
    MODEL_PATH = "models/yolov8n-pose.pt"  # 模型路径
    IMG_SIZE = 512         # 输入尺寸 512x512（比标准640更小，省算力）
    CONF_THRESH = 0.5      # 置信度阈值
    DEVICE = "cuda"        # GPU 加速
    HALF = True            # FP16 半精度
    INFERENCE_EVERY = 2    # 每2帧推理1次
```

### TensorRT 加速

如果用的是 NVIDIA 显卡，可以开启 **TensorRT**——NVIDIA 专门做的 AI 推理加速器。

原理：把 `.pt` 模型文件转换成 `.engine` 格式，针对你的 GPU 型号做专门优化。

**效果：推理速度大约提升 2 倍。**

```python
# 用法：一行代码，首次运行会自动转换
model = YOLO("models/yolov8n-pose.engine")  # .engine 替代 .pt
```

> 打个比方：`.pt` 是通用食谱，什么锅都能用；`.engine` 是针对你家这口锅专门调过的食谱，用起来更快更好。

### GPU 预热

第一次推理特别慢（比后续慢 50ms），因为 CUDA 要初始化、分配显存。

我们在**启动时先跑一次空推理**，把初始化的延迟提前消化掉：

```python
# detection/yolov8_security.py:1302-1307
dummy = np.zeros((512, 512, 3), dtype=np.uint8)  # 一张全黑的假图
model(dummy, imgsz=512, device="cuda", half=True, verbose=False)
print("GPU 预热完成")
```

> 就像运动员比赛前先跑两圈热身，真正开始检测时第一帧就不会卡顿。

### thresholds.json 动态加载

检测阈值（比如跌倒的长宽比阈值、打架的距离阈值）存在一个 `thresholds.json` 文件里。改这个 JSON 文件，**立即生效，不用重启程序**。

```json
{
  "fall_aspect_ratio": 0.7,
  "fight_distance": 0.15,
  "gather_threshold": 3,
  "gather_duration_sec": 3.0
}
```

> 运维人员直接改文件就能调灵敏度，不需要懂代码、不需要重启服务。

## 5. 总结（0.5min）

三句话总结：

1. **为什么微调**——预训练模型通用，监狱场景特殊（统一服装、光线差、行为特殊），微调让它更适应
2. **怎么加速**——TensorRT 翻倍提速、GPU 预热消除首帧卡顿、FP16 半精度省内存
3. **怎么对比**——四代算法同屏对比，实时训练曲线可视化，一目了然
