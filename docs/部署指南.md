# YOLOv8 安全监控系统 - 阿里云部署指南

## 快速部署步骤

### 1. 在本地打包应用程序

双击运行 `build_exe.bat`，等待打包完成。
打包完成后，会在 `dist\YOLOv8_Security_Monitor\` 目录生成可执行文件。

### 2. 上传到阿里云服务器

将 `dist\YOLOv8_Security_Monitor\` 整个文件夹复制到阿里云服务器E盘：
```
E:\YOLOv8_Security_Monitor\
```

### 3. 服务器环境配置

#### 3.1 安装依赖（如果使用GPU）
- CUDA Toolkit 11.8+: https://developer.nvidia.com/cuda-downloads
- cuDNN 8.6+: https://developer.nvidia.com/cudnn

#### 3.2 配置阿里云安全组
登录阿里云控制台，添加安全组规则：
- 协议类型: TCP
- 端口范围: 5000/5000
- 授权对象: 0.0.0.0/0 (或指定IP)

#### 3.3 配置Windows防火墙
```powershell
# 以管理员身份运行PowerShell
New-NetFirewallRule -DisplayName "YOLOv8 Security" -Direction Inbound -Protocol TCP -LocalPort 5000 -Action Allow
```

### 4. 启动服务

在服务器上，进入 `E:\YOLOv8_Security_Monitor\` 目录，双击运行 `start_server.bat`

等待显示：
```
Running on http://0.0.0.0:5000
```

### 5. 访问系统

打开浏览器，访问：
```
http://你的服务器公网IP:5000
```

## 后台运行（可选）

使用 NSSM 将程序注册为Windows服务：

```batch
# 1. 下载 NSSM: https://nssm.cc/download
# 2. 安装服务
nssm install YOLOv8Security "E:\YOLOv8_Security_Monitor\YOLOv8_Security_Monitor.exe"

# 3. 启动服务
nssm start YOLOv8Security

# 4. 设置开机自启
nssm set YOLOv8Security Start SERVICE_AUTO_START
```

## 故障排查

### 端口被占用
```powershell
# 查看5000端口占用情况
netstat -ano | findstr :5000

# 结束占用进程
taskkill /PID <进程ID> /F
```

### 无法访问
1. 检查阿里云安全组是否开放5000端口
2. 检查Windows防火墙是否允许5000端口
3. 确认app.py中host设置为'0.0.0.0'
4. 检查服务器是否有公网IP

### GPU不可用
1. 检查NVIDIA驱动是否安装
2. 检查CUDA和cuDNN版本是否匹配
3. 查看程序启动日志中的CUDA信息

## 注意事项

1. **模型文件**: 首次运行需要下载YOLOv8模型，请确保服务器可以访问互联网
2. **存储空间**: 检测结果和录像会占用磁盘空间，定期清理
3. **内存使用**: 监控程序会占用较多内存，建议服务器至少8GB内存
4. **带宽**: 视频流会消耗带宽，根据实际需求调整

## 更新部署

如需更新程序：
1. 停止当前运行的服务
2. 替换 `YOLOv8_Security_Monitor.exe` 文件
3. 重新启动服务

数据文件（data、videos、results目录）可以保留，不会被覆盖。
