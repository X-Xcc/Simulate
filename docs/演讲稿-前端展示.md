### 4 张指标卡片

Dashboard 顶部是 4 张卡片，分别显示今日的**打架、跌倒、离岗、聚集**事件数：

```tsx

// 每张卡片：图标 + 标题 + 今日数量 + 较昨日增减
const cards = [
  { title: "打架事件", value: stats.fighting_today, icon: Swords, color: "red" },
  { title: "跌倒事件", value: stats.fall_today, icon: PersonStanding, color: "orange" },
  { title: "离岗事件", value: stats.absent_today, icon: UserX, color: "yellow" },
  { title: "人员聚集", value: stats.gather_today, icon: Users, color: "purple" },
];

// 渲染
{cards.map(card => (
  <div className="stat-card" style={{ borderLeft: `4px solid ${card.color}` }}>
    <card.icon size={20} />
    <span>{card.title}</span>
    <h2>{card.value}</h2>
    <p>{card.diff > 0 ? `↑${card.diff}` : `↓${abs(card.diff)}`} 较昨日</p>

```

> 四种颜色、四种图标，一眼就知道今天哪种行为最严重。

### Prison3D 3D 监区热力图

这是技术含量最高的组件——**在浏览器里画了一个 3D 监狱模型**：

```tsx
// web/src/components/Prison3D.tsx — 12 栋建筑
const BLDGS = [
  { id: "A", name: "一监区监舍楼", intensity: 82, alerts: 18 },
  { id: "B", name: "二监区监舍楼", intensity: 45, alerts: 7 },
  // ... 共 12 栋
];

// 危险程度 → 颜色映射
function heatColor(t: number) {
  if (t > 0.65) return lerp(青色, 红色, (t - 0.65) / 0.35);  // 高危 → 红
  return lerp(青绿, 青色, t / 0.65);                          // 安全 → 青绿
}
```

> intensity 82 的楼 = 红色（危险），intensity 10 的楼 = 青绿色（安全）。建筑内部有发光粒子浮动，报警越多粒子越多。还有一道蓝色光幕持续扫描——像科幻电影里的安全扫描。

鼠标可以拖拽旋转整个模型。这不是效果图——**实时数据驱动**。

## 5. 技术亮点（2min）— 详细讲

### API 缓存层：30s 内存缓存 + 并发请求去重

```tsx
// web/src/lib/api.ts — cachedFetch 核心逻辑
async function cachedFetch<T>(path: string, fetchFn: () => Promise<T>): Promise<T> {
  const entry = cache.get(path);
  // 1. 缓存命中: 30秒内直接返回
  if (entry && Date.now() - entry.ts < 30_000) return entry.data;

  // 2. 去重: 相同请求正在飞行中，复用同一个 Promise
  if (pending.has(path)) return pending.get(path);

  // 3. 发起新请求
  const promise = fetchFn().then(data => {
    cache.set(path, { data, ts: Date.now() });
    pending.delete(path);
    return data;
  });
  pending.set(path, promise);
  return promise;
}
```

> 这个设计解决了两个问题：
> - **缓存**：30 秒内同一接口不重复请求，减少 80% 的网络开销
> - **去重**：10 个组件同时请求 `/api/stats`，实际只发 1 次 HTTP 请求，10 个组件共享同一个结果
>
> GET 请求自动缓存，POST/PUT/DELETE 自动失效缓存。

### SSE 单例：1 个 EventSource 承载 5 种事件

```tsx
// web/src/lib/api.ts — 引用计数的单例模式
const sseSubscribers = new Map<string, Set<SseCallback>>();
let sseEventSource: EventSource | null = null;
let sseSubscriberCount = 0;

function ensureSseConnection() {
  if (sseEventSource) return;  // 已有连接，复用

  const es = new EventSource("/api/sse/stream");
  es.addEventListener("cameras", (e) => dispatch("cameras", e));
  es.addEventListener("alerts", (e) => dispatch("alerts", e));
  es.addEventListener("system_metrics", (e) => dispatch("system_metrics", e));
  es.addEventListener("audit_logs", (e) => dispatch("audit_logs", e));
  es.addEventListener("camera_stats", (e) => dispatch("camera_stats", e));

  es.onerror = () => {
    es.close();
    setTimeout(() => ensureSseConnection(), 3000);  // 3秒断线重连
  };
}
```

> **引用计数单例**：
> - 第一个组件订阅 → 自动创建连接
> - 后续组件订阅 → 复用同一个连接
> - 所有组件取消订阅 → 自动关闭连接
> - 断线 → 3 秒后自动重连
>
> 不用操心连接的生命周期，订阅就用，取消就走。
