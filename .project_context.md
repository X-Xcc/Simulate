# YOLOv8 Security System - Full Project Context

## Overview
AI-based security monitoring system using YOLOv8 pose estimation for behavior detection (fall, fight, fatigue, leave-post) + Qwen VL for visual analysis. Spring Boot backend serves a Thymeleaf dashboard. Pure file-based data — no database.

## Tech Stack
- **Backend**: Spring Boot 3.2.5, Java 17, Maven, WAR packaging
- **Frontend**: Thymeleaf template (`templates/index.html`), static `frontend/index.html`
- **AI**: YOLOv8n-pose (ultralytics), Qwen2.5-VL-7B-Instruct (optional)
- **Python**: 3.10+, .venv virtual env
- **Deploy**: Docker Compose + Nginx (HTTPS), or native via `start.bat`
- **DB**: None — detection JSONs stored in filesystem

## Data Flow
1. `yolov8_security.py` runs YOLOv8n-pose on video frames → writes JSON to `DATASET_DIR` every 5 seconds
2. Python sends raw frames via POST to `http://127.0.0.1:5000/yolov8-security/api/update_frame`
3. Spring Boot `DetectionService` reads JSON files from `app.file.upload-dir`
4. Frontend polls `/api/recent` (detections) and `/api/stats` (aggregates)
5. Video stream via SSE from `/api/stream`

## Fixes Applied (2026-04-26)
1. **Port mismatch fixed**: `start.bat` had `--server.port=8080` override, but Python posts to port 5000. Removed override — backend now runs on port 5000 matching config.
2. **Context-path alignment**: `application.properties` and `application-docker.properties` both set `context-path=/yolov8-security` to match Python `WEB_SERVER_URL` and Nginx proxy.
3. **Path verification**: `application.properties` uses `${user.dir}/data` — since `start.bat` runs from `BACKEND_DIR` (`PROJECT_ROOT/backend`), this resolves to `PROJECT_ROOT/backend/data` matching Python's `DATASET_DIR`.

## Remaining Issues
- `start.bat` still references port 8080 in echo/help text (cosmetic only)
- `mysql-connector-j` in pom.xml is unused — no DB exists
- `application.properties` `static-locations` uses `file:./data/` (relative) — works when CWD is `BACKEND_DIR`

## API Endpoints
| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/status` | System status (monitor running?, model info) |
| GET | `/api/recent` | Last 20 detection records |
| GET | `/api/stats` | Aggregated stats (person counts, action counts) |

| GET | `/api/stream` | SSE video frame stream |

## Auth
- `AuthFilter` checks `X-API-Key` header against `app.api-key` property
- Skips if `app.api-key` is null/blank
- `application.properties` sets `app.api-key=${APP_API_KEY:change-me-in-production}`
