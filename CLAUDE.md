## Project Overview

监狱智能行为分析系统 (Prison Intelligent Behavior Analysis System) — real-time security monitoring using YOLOv8 pose estimation. Detects falls, fighting, fatigue, eye fatigue, and guard absence in prison environments.

## Architecture

Two-component system:

### 1. Python AI Detection (`ai-models/yolov8_security.py`)
- YOLOv8n-pose model for real-time pose estimation
- Detects: 跌倒 (fall), 打架 (fighting), 疲劳 (fatigue/posture), 眼疲劳 (eye fatigue via EAR), 离岗 (leave post)
- OpenCV-based video processing with UI overlay (OpenCV window)
- Streams frames to Java backend via HTTP POST (`/api/update_frame`)
- Saves detection JSON + frame images to `backend/data/` directory
- Modules: `DetectionModule` (behavior detection), `DataSaver`, `AlertManager`, `UIManager`
- Optional Qwen2.5-VL service (`ai-models/qwen_vl_service.py`) for LLM-based scene analysis

### 2. Java Spring Boot Backend (`backend/`)
- Spring Boot 3.x, Java 17, Maven (WAR packaging)
- Serves web dashboard (Thymeleaf template at `src/main/resources/templates/index.html`)
- REST API endpoints under `/yolov8-security/`:
  - `GET /api/stats` — detection statistics + behavior counts (polled every 2s)
  - `GET /api/images` + `GET /api/images/{name}` — screenshot listing/serving
  - `DELETE /api/delete_all_images` — bulk delete detection data
  - `GET /api/model_info` — YOLO model quantization info
  - `GET /api/ai/status` — Qwen2.5-VL status
  - `POST /api/open_folder` — open screenshots/videos folder on server
  - `GET /video_feed` — MJPEG video stream
- Services: `DetectionService` (cached file scanning), `ModelInfoService`, `PythonScriptService`, `QwenVLService`
- Filters: `AuthFilter` (token-based API auth), `RateLimitFilter`, `CorsConfig`
- Nginx reverse proxy (`nginx/nginx.conf`), Docker deployment

### Communication Flow
```
Camera → YOLOv8 Pose (Python) → saves JSON/images to backend/data/
                              → HTTP POST frames to Java backend
                              → Java backend serves REST API + MJPEG stream
                              → Web dashboard polls /api/stats every 2s
```

## Key Implementation Details

- **DetectionService** uses a flat-file scan approach (no DB) — reads JSON files from `data/` directory, parsed via Jackson `ObjectMapper`. Caches results with `invalidateScanCache()` for refresh.
- **StatsResponse** aggregates behavior counts via `BehaviorCounts` POJO (fall, fight, absent, fatigue fields). Recent detections capped at 50, all detections at 200.
- **Video streaming** uses MJPEG via `VideoStreamController` — reads JPEG frames from disk, writes multipart/x-mixed-replace response.
- **Auth** is token-based via `AuthFilter` — checks `Authorization` header against `app.auth.token` property.
- **Rate limiting** implemented in `RateLimitFilter` using token bucket per IP.
- **Python Config class** (line 56 in yolov8_security.py) centralizes all parameters: model path, detection thresholds, alert cooldown, UI colors, save intervals.

## Model Files

- YOLOv8n-pose: `models/yolov8n-pose.pt` (pose estimation, ~5MB ONNX/PyTorch)
- Qwen2.5-VL-7B: Optional multimodal model for advanced scene analysis (~14GB)

## Config

- Python: `Config` class in `ai-models/yolov8_security.py` (line 56)
- Java: `application.properties` + `AppConfig.java`
- Web server URL: env var `WEB_SERVER_URL` (default: `http://127.0.0.1:5000/yolov8-security`)

## Important Notes

- No database — all detection data is stored as flat JSON files in `backend/data/`
- The Python script reads from a video file (`videos/test_video.mp4`), not a live camera by default
- Java tests use JUnit 5 with `@TempDir` for isolated file I/O testing — no mocking framework
- The web UI is a single Thymeleaf template with embedded CSS/JS using Chart.js
